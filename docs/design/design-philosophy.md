# 设计哲学 · Design Philosophy

> 可交互原型见 [`assets/product-ui-prototype.html`](../assets/product-ui-prototype.html)。

## 核心命题:把"创造"变成"审核"

求职工具让用户累的根源,是每一步都是高认知负担的**创造任务**。Relay 的核心交互理念只有一句话:

> **AI 先做,用户只做两个动作——选择(yes/no)和微调(edit)。**

| 用户的旧任务(累) | Relay 转化后(轻松) |
|---|---|
| 填 profile 表单 | 上传简历 → AI 解析 → 审核 |
| 写 cover letter | AI 生成 → 改几个字 |
| 判断职位是否值得投 | AI 算 match% → 一眼筛选 |
| 定制简历 | AI 定制 → 确认 |
| 填表 | AI 填好 → 检查高亮项 |

步骤没有真的变少,而是从用户的大脑搬到了 AI 这边,用户只剩下最轻的决策动作。这就是"步骤少但超级有效"的底层逻辑。

## 六条设计原则

### 1. 0 配置启动
唯一的 onboarding 动作:上传简历。AI 解析出工作经历、技能、方向、匹配职位。对比传统工具填 20+ 字段,这里是"上传 1 个文件"。

### 2. 3 次点击投递
从看到职位到完成提交,控制在 3 次点击以内。

### 3. AI 先做,用户后审
永不让用户面对空白。简历摘要、求职信、表单答案全部预生成,用户的工作降级为"扫一眼 + 改两个字"。

### 4. 一屏一焦点
每屏只有一个主操作(深色大按钮),消除决策疲劳。

### 5. 上下文浮现,而非菜单导航
不让用户在菜单里找功能。用户在什么状态,系统就主动提供该状态下最相关的 action。例:有面试的职位卡片自动浮出"准备面试"按钮——面试准备不是要主动找的功能,而是在你需要它的那一刻出现。

### 6. 颜色即语言
一套贯穿全产品的色彩语义,用户不用读说明就懂:
- **深靛蓝** = 你的主操作
- **珊瑚橙** = AI 生成的、需要你过目的内容
- **薄荷绿** = 安全 / 成功 / 高匹配

## 签名设计:AI 内容的统一视觉标记

整个产品最重要的记忆点:**所有 AI 生成的内容都用珊瑚橙标记**(带 "AI 生成·草稿" 的 chip)。

这一个视觉语言解决了核心的信任问题——用户一眼就知道"哪些要重点检查",不用通读全部。橙色出现的地方 = "停一下,这是 AI 帮你写的,过目一下"。

## 视觉方向:平静的掌控感

刻意避开 AI 设计的三个套路(米色+衬线、纯黑+荧光、报纸风)。

求职是个充满焦虑的过程。大多数求职工具的设计是"忙碌的、信息密集的"(像招聘网站那样密密麻麻),这本身就是焦虑的来源。Relay 的差异化是**"平静的掌控感"**——像一个高端私人助理,把混乱整理得井井有条。

所以用了:大量留白、柔和的纸白背景、克制的靛蓝主色,而非密集的信息流。

## 文案原则

- **用户视角**:用人能控制和识别的词命名,不暴露系统实现。
- **主动语态**:按钮说清楚点了会发生什么。"审核无误,投递" 而非 "提交"。
- **一致词汇**:一个动作从头到尾同名。说 "投递" 的按钮,产生说 "已投递" 的提示。
- **空状态是邀请**:空屏是行动的邀请,不是装饰。
- **错误给方向**:说清出了什么问题、怎么修,用界面的语气,不道歉、不含糊。
