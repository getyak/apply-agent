# 产品规格 · Product Spec

6 大核心功能的详细定义、优先级与边界。

## 优先级矩阵

| # | 功能 | 复杂度 | ToS 风险 | 用户价值 | 阶段 |
|---|------|--------|---------|---------|------|
| 1 | 简历管理(CRUD + 版本) | ⭐ | 无 | 高 | **MVP** |
| 2 | AI 简历优化 | ⭐⭐ | 无 | 高 | **MVP** |
| 3 | JD 定制简历 | ⭐⭐⭐ | 无 | 高 | **MVP** |
| 4 | JD 面试模拟 | ⭐⭐⭐ | 无 | 高 | **MVP** |
| 5 | 客户端半自动投递 | ⭐⭐⭐⭐ | 低 | 高 | **MVP**(扩展) |
| 6 | 日常趋势分析 | ⭐⭐⭐ | 低 | 中 | **MVP**(基础版) |

> 核心决策:MVP 做全 6 个功能,但投递只做**客户端半自动**(用户审核后亲自提交),不做服务器端全自动。

---

## 功能 1:简历管理

**用户故事**
- 上传/粘贴现有简历 → 系统解析为结构化 [JSON Resume](https://jsonresume.org/)
- 在 UI 中编辑各 section,自动保存版本(timestamp + diff)
- 可恢复旧版本或对比两版

**关键点**
- 解析用 LLM(via OpenRouter,DeepSeek V4 Flash),输出符合 JSON Resume schema
- 版本不覆盖,保留完整历史
- 这是整个系统的数据脊骨——"写一次,映射到多处"

---

## 功能 2:AI 简历优化

**用户故事**
- 点"优化",可选提供 JD
- AI 生成改进建议:强化 bullet、添加量化指标、改进用词
- 展示原文 vs 优化版 diff,用户选择接受/拒绝

**Prompt 纪律**
> "诚实重述,不要编造。强化已有成就的表达,不虚构经历。"

**质量指标**
- 语义相似度(优化后是否仍真实)
- 用户是否大改(改得越少说明质量越高)

---

## 功能 3:JD 定制简历

**用户故事**
- 提供一个 JD(粘贴或链接提取)
- AI 读 JD → 识别关键技能 → 重排/强调 base 简历中相关项 → 标记技能缺口
- 用户 review,可微调,保存为 tailored 版本

**输出**:针对该职位的简历 + cover letter + 表单答案草稿,打包成一个"投递包"。

---

## 功能 4:JD 面试模拟

**用户故事**
- 选职位或粘贴 JD
- AI 生成可能的面试问题(技术 + behavioral + situational)
- 用户逐个作答(文本或语音转文字)
- AI 评估回答 + 建议改进 + 给参考答案
- 展示该公司/角色的面试 tips

**数据飞轮入口** ⭐
- 用户记录"我在 X 公司被问了什么、怎么答的、结果如何"
- 系统按 公司/角色/阶段/问题 结构化存储
- 用户 opt-in 后,聚合成众包面试题库("X 公司后端最常问的 10 题")
- **这是 Relay 最 sticky 的功能** — 个人面试历史是难以迁移的资产

## 功能 5:客户端半自动投递 ⭐

**这是市场差异化的核心,也是最需要谨慎设计的部分。**

**用户故事**
- 浏览器扩展检测到职位申请页 → 浮现"一键填充"
- AI 填好所有字段(简单字段本地填,复杂字段/开放题走云端 LLM)
- 用户审核(AI 生成的部分高亮标记)→ **用户亲自点 submit**

**三层字段处理**
1. **~70% 简单字段**(name/email/phone)→ 本地规则填充,零成本零延迟
2. **~25% 复杂/陌生字段** → 云端 LLM 智能映射
3. **~5% 开放题** → 云端 LLM 基于 JD+简历生成草稿

**红线**
- 不做 LinkedIn/Indeed/Boss直聘 登录态自动投递
- 不代理发送邮件
- 不自动点 submit——最终提交永远是用户的动作
- 提供 dry-run 模式(生成但不提交,用于测试)

详见 [客户端投递方案](architecture/client-side-delivery.md)。

## 功能 6:日常趋势分析

**用户故事**
- 每天收到"市场快照":昨日新增职位、热门技能排名、热门角色、薪资范围、**你缺少的 trending 技能**

**数据源(无 ToS 风险)**
- Greenhouse / Lever / Ashby 的 public 职位 feed(无认证)
- 数据量已足够看趋势

**关键:Actionable,不只是统计**
> 不是 "TypeScript 上涨 8%",而是 "你有 Python、TypeScript,但缺 Rust(↑12%)。学了 Rust 预计多匹配 47 个岗位。"

这个"if you learn X, you'd unlock Y" 的心理钩子非常强。

## 不在 MVP 中(Phase 2+)

- ❌ 任何形式的服务器端全自动投递
- ❌ 桌面 App + CDP(browser-use/Stagehand,power user 功能,见路线图)
- ❌ 高级趋势分析(付费数据源)
- ❌ 真人面试教练(marketplace)
