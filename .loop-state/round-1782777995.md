## Honest caveats (read these before celebrating)

The probe verifies each rubric item at the **highest layer reachable in this
sandbox**. Specifically:

- Items 5 (envelope), 6 (i18n), 7 (trace continuity) were verified
  **agents-internal** only. The cross-LAYER guarantee ("web → api → agents
  logs share one traceId" from `error-handling.md` § 5) cannot be re-verified
  in this run because the api gateway can't `bun install` (lockfile pinned
  to a blocked registry) and the web layer has the same problem. The
  cross-layer wiring itself is in place — `api/src/middleware/trace-id.ts`
  generates / honours `X-Trace-Id`, `web/src/lib/api.ts` reads
  `x-trace-id` off responses into `ApiError.traceId`, and the agents
  middleware (verified live this run) echoes it — but a true end-to-end
  trace round-trip will only be re-provable in a round where docker pulls
  succeed.
- Idempotency (item 8): in hermetic mode `_create_application_draft` returns
  the explicit `application_id` unchanged when `RELAY_PG_DSN` is absent (it's
  designed to no-op out), so the stable-id check is structurally correct but
  the DB UNIQUE(application_id) protection is not exercised. A round with
  real PG would close that gap.
- Cost guard (item 4): probe constructs an `open_tally()` directly and feeds
  `post_model_hook` a known-overrun cost — that's the exact production
  call-site that LangGraph's post_model_hook drives, so this is a real
  trigger of the cost limit, but it bypasses the LLM call site that would
  normally drive the tally (no OpenRouter access).
